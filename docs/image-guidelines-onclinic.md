# Гайдлайн роботи з фото · onclinic-web

Версія: 1.0 · Квітень 2026

---

## Принцип

Фото на сторінках клініки — тільки реальні фото від клініки.
Стокові фото заборонені — порушують довіру пацієнта і E-E-A-T.

---

## Типи фото на сторінках Онклінік

| Тип | Де використовується | Хто надає |
|-----|--------------------|-----------| 
| Фото операційної | Сторінки операцій (блок "Метод") | Олена Буцько |
| Фото обладнання | Сторінки операцій (картки обладнання) | Олена Буцько |
| Фото лікаря | Профілі лікарів, картки на сторінках | Олена Буцько |
| Фото фасаду/рецепції | Хаб клініки /kharkiv | Олена Буцько |

---

## Процес: від фото Олени до сторінки

```
1. Олена надає фото (Telegram або email)
2. Ігор завантажує фото в чат Claude (цей проект)
3. Claude:
   - Генерує ім'я файлу за стандартом
   - Генерує alt-текст (UA + SEO)
   - Вказує розміри для оптимізації
   - Генерує запис для MD-файлу
4. Ігор зберігає фото в assets/images/[папка]/
5. Cowork Dev вставляє в HTML з правильним alt і розмірами
```

---

## Стандарт іменування файлів

```
[клініка]-[місто]-[тип]-[деталь].[ext]

Приклади:
onclinic-kharkiv-operating-room.jpg
onclinic-kharkiv-laparoscopy-equipment.jpg
onclinic-kharkiv-doctor-afanasiev.jpg
onclinic-kharkiv-reception.jpg
```

Правила:
- Тільки латиниця, дефіс як роздільник
- Без пробілів, без кирилиці, без спецсимволів
- Розширення: `.jpg` для фото, `.webp` після конвертації
- Без порядкового номера якщо фото одне, з номером якщо кілька: `operating-room-01.jpg`

---

## Стандарт alt-тексту

```
[Що зображено] · [Клініка] · [Місто]

Приклади:
"Операційна зала ОН Клінік Харків · лапароскопічне обладнання Karl Storz"
"Лапароскопічна стійка Karl Storz в ОН Клінік Харків Левада"
"Хірург-гінеколог Афанасьєв І.В. · ОН Клінік Харків"
```

Правила:
- Українська мова
- До 125 символів
- Не починати з "Фото" або "Зображення"
- Включати назву клініки і місто
- Включати медичний контекст (що відбувається/що зображено)

## Запис у MD-файлі

Після обробки Claude додає в MD-файл блок:

```markdown
## Фото

### [Назва блоку де використовується фото]
- Файл: `onclinic-kharkiv-operating-room.jpg`
- Alt: "Операційна зала ОН Клінік Харків · лапароскопічне обладнання Karl Storz"
- Розміщення в репо: `assets/images/kharkiv/`
- Використовується в блоці: "Метод операції" (секція обладнання)
- Розміри: 1200×800px (оригінал), відображається 800×533px
- Статус: ⏳ очікуємо від Олени / ✅ є в репо
```

## Технічні вимоги до фото

| Параметр | Вимога |
|----------|--------|
| Формат оригіналу | JPG або PNG від клініки |
| Мінімальна ширина | 1200px |
| Цільовий формат | WebP (конвертує Cowork) |
| Якість WebP | 85% |
| Максимальний розмір файлу | 200KB після конвертації |
| `loading` атрибут | `lazy` для всіх крім першого фото на сторінці |
| `decoding` | `async` |

---

## Що робить Claude при отриманні фото

1. Визначає тип фото (операційна / обладнання / лікар / інше)
2. Генерує ім'я файлу за стандартом
3. Генерує alt-текст українською
4. Вказує де зберегти в репо
5. Генерує HTML-код для вставки:

```html
<img
  src="/assets/images/kharkiv/onclinic-kharkiv-operating-room.jpg"
  alt="Операційна зала ОН Клінік Харків · лапароскопічне обладнання Karl Storz"
  width="800"
  height="533"
  loading="lazy"
  decoding="async"
/>
```

6. Генерує запис для MD-файлу

## Список А для Олени — фото

Фіксувати запити на фото в Список А:

```
ФОТО ПОТРІБНІ:
□ Операційна зала (загальний вид з обладнанням)
□ Лапароскопічна стійка Karl Storz (крупний план)
□ Гістероскопічне обладнання
□ Фото фасаду філії Левада
□ Фото рецепції філії Левада
```

## Де зберігати фото в репо

```
onclinic-web/
└── assets/
    └── images/
        └── kharkiv/
            ├── doctors/          ← фото лікарів
            ├── equipment/        ← фото обладнання
            ├── operating-room/   ← фото операційної
            └── clinic/           ← фото фасаду, рецепції
```
